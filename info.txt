# Placement Preparation Tracker - Elite Edition
## Project Documentation

### 1. Project Overview
The Placement Preparation Tracker is a sophisticated, multi-user web application designed to help students and job seekers systematically track their placement preparation journey. It features an "Elite" design system with glassmorphism effects, smooth animations, and a professional slate color palette. The application supports detailed tracking of various preparation rounds (Aptitude, Technical, Coding, Interviews) and visualizes consistency using a Git-style heatmap.

### 2. Key Features
- **Elite UI/UX**: Modern, clean interface using Tailwind CSS with glassmorphism, floating animations, and a premium typography stack (Playfair Display & Plus Jakarta Sans).
- **Multi-User Support**: Secure authentication system allows multiple users to maintain private, isolated progress logs.
- **Visual Analytics**: 
  - Git-style contribution heatmap to track daily consistency.
  - Progress rings and percentage indicators.
  - Streak tracking (Current & Longest).
- **Comprehensive Tracking**:
  - **Round 1 (Aptitude)**: Topic-wise tracking (Quants, Logical, Verbal).
  - **Round 2 (Technical)**: Core subject mastery (OS, DBMS, CN, OOPs).
  - **Round 3 (Coding)**: DSA problem tracking (Easy, Medium, Hard).
  - **Round 4 (Interview)**: Mock interview logs and soft skills practice.
- **Responsive Design**: Fully responsive layout with mobile-optimized bottom sheets and adaptive grids.
- **Gamification**: Confetti celebrations upon completing daily goals and interactive motivation cards.

### 3. Technology Stack
**Frontend:**
- React.js (Vite)

### 5. Local Setup Instructions

**Prerequisites:**
- Node.js (v14+)
- MongoDB (Local or Atlas)

**Step 1: Backend Setup**
1. Navigate to the server directory:
   ```bash
   cd server
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env` file in the `server` directory:
   ```env
   PORT=5000
   MONGO_URI=mongodb://localhost:27017/placement-tracker
   JWT_SECRET=your_super_secret_key_change_this
   ```
4. Start the server:
   ```bash
   npm run dev
   ```

**Step 2: Frontend Setup**
1. Navigate to the root directory (if not already there):
   ```bash
   cd ..
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
   (Vite will typically run on http://localhost:5173 or 5174)

### 6. Deployment Guide (Public Use)

To deploy this application for public use, we will use a **MERN Stack** deployment strategy:
- **Database**: MongoDB Atlas (Cloud Database)
- **Backend**: Render, Railway, or Heroku
- **Frontend**: Vercel or Netlify

#### Phase 1: Database Deployment (MongoDB Atlas)
1. Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) and create a free account.
2. Create a new Cluster (Shared/Free tier).
3. In "Database Access", create a database user (username/password).
4. In "Network Access", allow access from anywhere (`0.0.0.0/0`).
5. Click "Connect" > "Connect your application" and copy the Connection String.
   - Replace `<password>` with your actual database user password.

#### Phase 2: Backend Deployment (Render)
1. Push your code to a GitHub repository.
2. Sign up on [Render](https://render.com/).
3. Click "New" > "Web Service".
4. Connect your GitHub repository.
5. Settings:
   - **Root Directory**: `server`
   - **Build Command**: `npm install`
   - **Start Command**: `node index.js`
6. **Environment Variables** (Advanced):
   - Add `MONGO_URI`: (Paste your Atlas connection string)
   - Add `JWT_SECRET`: (Set a strong secret key)
   - Add `PORT`: `5000` (or leave default, Render manages this)
7. Deploy. Once finished, copy the **Backend URL** (e.g., `https://my-api.onrender.com`).

#### Phase 3: Frontend Deployment (Vercel)
1. **Update API Base URL**:
   - In your frontend code (`src/context/AuthContext.jsx`, `src/components/Dashboard.jsx`, etc.), you are currently using `http://localhost:5000`.
   - **Best Practice**: Create a `.env` file in the root directory:
     ```env
     VITE_API_URL=https://my-api.onrender.com/api
     ```
   - Replace hardcoded URLs in your code with `import.meta.env.VITE_API_URL`.
   
2. **Deploy to Vercel**:
   - Go to [Vercel](https://vercel.com/) and sign up.
   - "Add New" > "Project".
   - Import your GitHub repository.
   - Settings:
     - **Framework Preset**: Vite
     - **Root Directory**: `./` (default)
     - **Build Command**: `npm run build`
     - **Output Directory**: `dist`
   - **Environment Variables**:
     - Add `VITE_API_URL`: `https://my-api.onrender.com/api` (The URL from Phase 2)
   - Click **Deploy**.

#### Phase 4: Final Configuration
1. Go back to your Backend (Render) Dashboard.
2. If you set up CORS in `server/index.js`, ensure it allows your new Vercel Frontend URL.
   - Update `server/index.js`:
     ```javascript
     app.use(cors({
       origin: ["https://your-frontend-app.vercel.app", "http://localhost:5173"],
       credentials: true
     }));
     ```
   - Redeploy Backend.

### 7. API Endpoints

**Auth Routes**
- `POST /api/auth/signup`: Register a new user.
- `POST /api/auth/login`: Authenticate user and get token.

**Data Routes** (Protected - requires Bearer Token)
- `GET /api/daylog`: Fetch logs for a specific date range.
- `POST /api/daylog`: Create or update a daily log.
- `GET /api/topics`: Fetch available topics for rounds.
